How it works

Figma → Plugin → Shareable link.

Three steps. No coding. Your devs get a polished, structured brief.

Design in Figma
Work in your Figma file as usual. Name your section frames clearly with a prefix of // — name the section as it is in the Theme to help the dev.
→
Tag in the plugin
Open the Handover plugin. Set each section to Existing, Modified, or Bespoke and add your notes.
→
Share the link
Hit "Open in Handover." Your devs get a clean, read-only brief — no Figma account needed. You can also export to ClickUp from there.
Setup guide

Up and running
in ten minutes.

No coding required. Just a few accounts, a couple of tokens, and a drag-and-drop. We'll walk you through every click.

Before you start, you'll need:
  • A free GitHub account (for saving your project data)
  • A Figma account (you already have this!)
  • A free Netlify account (for hosting your handover page)
Don't have a GitHub account? Head to github.com and sign up — it's free and takes 2 minutes. You won't need to write any code. We just use GitHub as a free, reliable place to store your handover data.
1
Create a GitHub token
~3 minutes · One-time setup

A "token" is like a password that lets the handover tool save and read your project data on GitHub. You only need to create it once.

  1. Log into GitHub, click your profile picture (top right), then go to Settings.
  2. Scroll all the way down in the left sidebar and click Developer settings.
  3. Click Personal access tokens → Tokens (classic).
  4. Click "Generate new token" → "Generate new token (classic)".
  5. Give it a name like Design Handover. Set expiration to "No expiration" (so you don't have to redo this).
  6. Under scopes, tick only the gist checkbox. Nothing else is needed.
  7. Click the green "Generate token" button at the bottom.
  8. Copy the token immediately and paste it somewhere safe (like Notes or a password manager). GitHub only shows it once!

3
Configure the web app
~1 minute · Paste your token
  1. Open your Netlify URL with ?setup=1 added to the end — e.g. your-site.netlify.app/design-to-dev-handover-v3-figma.html?setup=1. This bypasses the access gate and opens the settings panel automatically.
  2. Paste your GitHub token (from Step 1) into the GitHub Token field.
  3. For the Figma token: go to Figma → click your profile icon (top left) → Settings → scroll to "Personal access tokens" → type a name → hit Enter → copy the token.
  4. Paste the Figma token into the Figma Token field.
  5. Click Save. Your tokens are stored in the browser — you won't need the ?setup=1 param again after this. Just open your Netlify URL normally from now on.

4
Install the Figma plugin
~2 minutes · One-time install

The Figma plugin is what lets you tag frames and push them to the web app. Download the latest version below.

🔌
Figma Plugin
Design → Dev Handover  ·  latest
Download
View release notes →
  1. Unzip figma-plugin.zip to get a folder containing manifest.json, code.js, and ui.html. Save it somewhere you won't accidentally delete it.
  2. In Figma desktop app, go to Menu (hamburger icon, top left) → Plugins → Development → Import plugin from manifest…
  3. Navigate to the plugin folder and select the manifest.json file.
  4. The plugin is now installed. To run it: right-click the canvas → Plugins → Development → Design → Dev Handover.
✓ You should see the Kubix plugin panel open inside Figma with the yellow logo and your page's frames listed.
5
Connect the plugin to your web app
~1 minute · Final step!
  1. In the plugin, click the gear icon (top right) to open Settings.
  2. Paste your Netlify URL (from Step 2) into the "Handover App URL" field.
  3. Paste the full URL of your Figma file into the "Figma File URL" field. (Just copy it from your browser's address bar while the file is open.)
  4. If your web app already has a Gist ID (shown in the web app's sync modal), paste it into the "Gist ID" field. This links the plugin to the same project.
  5. Paste your GitHub token (same one from Step 1) into the "GitHub Token" field. This is needed for syncing.
  6. Go back to the main view. You should see your frames listed. Tag them, add notes, and hit "Open in Handover" to send everything to your web app!
🎉 You're all set! Your frames will appear in the web app with all their statuses and notes. Share the Netlify link with your developers and they'll see a clean, read-only handover brief.
